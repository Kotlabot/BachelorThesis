## Documentation of Genetic algorithm Optimizer
This section describes the workflow of the genetic algorithm, its components, and its parametrization for protein stability optimization. Detailed implementation documentation is available in the source code comments.

The algorithm randomly generates an initial population of individuals and evaluates them using a stability predictor. Based on their fitness values, individuals are selected for reproduction and a new generation is created using variation operators. This process is repeated iteratively until a termination criterion is met.

### Program Parameters
The algorithm takes several input parameters defined by the user in the parametrization file, including protein code, restricted sites, amino acids, number of individuals, and number of generations.  

- The protein code is a UniProt identifier of the protein of interest. Based on this identifier, the program downloads the protein sequence in FASTA format from UniProt database and the predicted protein structure in PDB format from the AlphaFold database. These files are then stored locally for further computation. As this parameter is essential, the program terminates with an exception if it is not provided. 

- The list of restricted sites contains positions in the protein which must remain unchanged. These typically include active or binding sites, where mutations could disrupt function or structure. Additionally, regions not relevant to stability, such as signal peptides, may also be excluded to reduce the search space. If no restricted sites are provided, the program raises a warning and continues without restrictions. 

- The list of amino acids defines which residues are allowed during mutation. In some applications, restricting the amino acid alphabet may be beneficial. If no list is provided, the program defaults to all 20 standard amino acids. Invalid amino acid codes result in termination with an exception. 

- The number of individuals and the number of generations are standard genetic algorithm parameters defining the population size and the number of evolutionary iterations. In case these parameters are not provided, the program raises a warning and defaults to 100 individuals and 100 generations. 

### Genetic Algorithm Components
- **Representation of individuals**: Each individual is represented as a list of pairs (tuples) where each pair consists of a mutation position and the substituted amino acid. This representation was chosen because every individual contains direct information about number of mutations it consists, and at which position and what amino acid it differs from the wild-type. To avoid structural disruption and excessive growth of the search space, the number of mutations per individual was implicitly limited to a maximum of four. 
- **Initialization**: Before the evolutionary loop begins, the program initializes the population randomly. Each individual is generated by selecting a random allowed position and a random amino acid, excluding the wild-type amino acid at that position. As a result, the initial population consists of single-point mutants. 
- **Fitness Function**: The fitness function is based on the external stability predictor FoldX. For each individual, the program generates a mutation list and passes it together with the wild-type structure to FoldX. FoldX evaluates the mutation by computing the difference in folding free energy ($\Delta \Delta G$) between the mutant and wild-type structures, used by the program as the fitness criteria for selection. If a FoldX evaluation fails, the individual is assigned a large penalty fitness value, effectively removing it from selection. FoldX outputs can be slightly different through multiple runs. Therefore, for a higher accuracy, each individual is evaluated three times, and the average $\Delta \Delta G$ value is used as the fitness score, because repeated evaluation reduces noise. Since lower $\Delta \Delta G$ values correspond to higher stability, the program aims to minimize the fitness scores. FoldX was selected due to its widespread use and relatively high accuracy in protein stability prediction.
- **Selection**: Parent selection is implemented using tournament selection with a tournament size of three. In other words, for each selection three individuals from the population are randomly chosen to be part of the tournament. From these three individuals, with 60\% probability the individual with the best (lowest) fitness is selected. Otherwise, a random individual from the tournament is chosen. This approach balances selection pressure with population diversity and helps prevent premature convergence. Survivor selection is implemented using age-based selection, when offspring replace parents in every generation. In addition, elitism is applied by carrying over 5\% of the best-performing individuals unchanged to the next generation. This ensures that high-quality solutions are preserved throughout the evolutionary process. 
- **Variation Operators**: Offspring are created from parents using several variation operators, applied with predefined probability. The operator probabilities were chosen empirically to balance exploration of new regions of the search space and exploitation of already discovered high-quality solutions. 
1. With 5\% probability, the offspring is created from its parent by removing one of its mutations. 
2. With 25\% probability, the crossover operator is applied, which combines good solutions together. In this case, the second parent is selected through another tournament selection, and then the offspring is created by inheriting one mutation from each parent.
3. With a 40\% probability, the offspring is created by inheriting all mutations from its parent and adding one more random mutation. This operator extends promising solutions to explore further improvements. 
4. Finally, with a 30\% probability, the offspring is created from parent by changing one of its mutations. In this case, one mutation from parent is altered either by changing its position (40\% probability) or amino acid (60\% probability). This operator promotes exploration of new regions of the search space. 
- **Termination and Outputs**: The algorithm terminates after a predefined number of generations. After completion, several outputs are generated for further analysis. First output is a text file which contains input parameters, the five best individuals from each generation along with their fitness value and the ten best individuals from the last generation along with their fitness values, where the first one is the absolute best individual found through the evolution. Second output is a PDB file produced by FoldX, three-dimensional structure of the best individual found, which can be later visualized to inspect the introduced mutations. Third output is a plot showing the fitness of the best and average fitness values across generations.  
- **Parallelization**: Fitness evaluations are parallelized using the multiprocessing module, allowing multiple individuals to be evaluated simultaneously on separate CPU cores. To support parallel execution and avoid file conflicts, each evaluation is performed in a separate working directory.
